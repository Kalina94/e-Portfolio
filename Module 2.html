Launching into Cyber Security

Engaging in debate to identify contemporary cyber security-based challenges and the ability to design appropriate solutions.

Learning Outcomes

At the end of this module, I shall be able to:

  • identify and explain the architecture, structure and functionality of basic components of a computer system, considering Cyber Security issues
  • evaluate critically implications of the key vulnerabilities and threats of software and network security and approaches to mitigate these issues
  • appraise critically and apply the concepts and principles of secure object-oriented design to facilitate business security-based decisions
  • evaluate critically the solutions developed to solve/mitigate these security issues.

Feedback Obtained

The report is well written with providing detailed benefits to be derived from ASMIS when installed. You employed the STRIDE modelling technique to identify potential vulnerabilities. The components of the STRIDE technique are well explained. The UML diagrams employed clearly demonstrate how each of the threat identified is likely to manifest. Firewall, encryption, 2 factor-authentication and login management are proposed as potential solutions to the attacks. The report could benefit from critical evaluation of the proposed solutions. The advantages were mentioned but it would be to also point out some disadvantages associated with the solutions as well. It is unclear how some of the attacks identified by the STRIDE technique were being addressed. An example is the Repudiation. Also, firewall was mentioned as a possible solution. However, it remains unclear the type of firewall being proposed. It is also unclear why “person” class and the “doctor” class are connected in the class diagram. The doctor has no direct interaction with the person. There references used in the report could better as they heavily relied on non-peer-reviewed articles and blogs. Also you have your word-count is excessively high above the acceptable margin. Overall, some good work done. To improve your grades further, do take note of the developmental areas highlighted above.

The code has an implementation of a solution to mitigate false login attacks. The solution is an implementation of a one-time password (OTPs) using a 2FA technology. The implementation employed Python, python-based Flask framework and Twilio API. The use of these technologies to secure user login details is well justified.

Even though documentation was clear to follow and the implementation logic was sound, the implementation did not ensure good password and username policies. For example, minimum length the format of acceptable passwords. These would have strengthened the security further and criticality of your assessment of the security and implementation strategy.
Structure and Presentation (as detailed in the assessment guidance): 30%
